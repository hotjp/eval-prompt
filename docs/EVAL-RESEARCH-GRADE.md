# Eval Research Grade：论文级评估基础设施设计

> 目标：不是做一个"更好看的打分工具"，而是建立一套**引用级（citation-worthy）**的 Prompt & Agent 评估方法论。

---

## 一、为什么"烂大街的量表"不行

当前市面上的 LLM 评估工具（包括我们现在的设计）普遍存在三个致命缺陷：

1. **指标无出处**："创造性 85 分"——这个指标定义在哪篇论文里？测试者之间的一致性（Inter-annotator Agreement）是多少？没人知道。
2. **分数不可复现**：同样的 Prompt 今天跑 82 分，明天跑 79 分，差异是噪声还是信号？没有统计检验，无法区分。
3. **绝对分数无意义**："82 分"是什么意思？80 分算好还是坏？科研界的做法是**成对比较 + 统计显著性**，而不是绝对打分。

**科研级的核心原则**：
- 每个指标都有论文支撑
- 每次结论都附带置信区间和 p-value
- 评估过程可复现、可审计
- 评估者（不管是人还是模型）的一致性被量化

## 二、指标体系：每颗钉子都有论文锤子

### 2.1 核心评估范式：成对比较（Pairwise Comparison）

**论文基础**：Thurstone, 1927 (Law of Comparative Judgment)；Bradley-Terry, 1952

**为什么不用绝对打分？**
- 人类（和 LLM）在给绝对分数时一致性很差（Kappa < 0.4）
- 但做"A 还是 B 更好"的判断时一致性高得多（Kappa > 0.7）
- 绝对分数会受锚定效应影响，成对比较不会

**我们的实现**：
```
不是：Prompt A 得了 82 分
而是：Prompt A vs Prompt B，在 95% 置信水平下，A 显著优于 B（p=0.003，效应量 d=0.68）
```

**界面呈现**：
- 每个版本不是显示"分数"，而是显示 **ELO 排名**
- 版本间的连线标注：**显著优于 / 无显著差异 / 显著劣于**
- 置信区间可视化：误差棒图

### 2.2 指标一：语义相似度（BERTScore）

**论文**：Zhang et al., *BERTScore: Evaluating Text Generation with BERT*, ICLR 2020

**原理**：
- 不比较 token 字符串（BLEU 的问题），比较 token 在 BERT 嵌入空间中的余弦相似度
- 计算 Precision（生成文本中有多少被参考文本覆盖）、Recall（参考文本中有多少被生成文本覆盖）、F1

**适用场景**：
- 有明确 Ground Truth 的生成任务（摘要、翻译、问答）
- 替代我们现在的 "Deterministic Score"

**科研级要求**：
- 指定使用的预训练模型（bert-base-chinese / RoBERTa / 领域特定模型）
- 记录层数（通常用最后四层平均）
- 报告 Rescaling 方法（baseline 校正）

**界面**：
```
BERTScore
├─ Precision: 0.84  [████████░░]  CI: [0.81, 0.87]
├─ Recall:    0.79  [███████░░░]  CI: [0.76, 0.82]
└─ F1:        0.81  [████████░░]  CI: [0.79, 0.83]

模型: bert-base-chinese | 层: 9-12 | Baseline: idf | 论文: Zhang et al. ICLR 2020
```

### 2.3 指标二：基于 LLM 的 Criteria 评估（G-Eval）

**论文**：Liu et al., *G-Eval: NLG Evaluation using GPT-4 with Better Human Alignment*, EMNLP 2023

**原理**：
- 用 LLM（GPT-4 或自研评估模型）根据自定义 criteria 打分
- 关键技巧：Chain-of-Thought —— 先让模型写评估理由，再出分数
- 比直接打分与人类判断的一致性高 30%+（Pearson r 从 0.45 提升到 0.78）

**替代我们现在的 "Rubric Score"**：
- 现在的 Rubric 是硬编码的 check list，太僵化
- G-Eval 允许自然语言描述的 criteria，且支持复杂推理

**科研级要求**：
- 温度设为 0（保证可复现）
- 多次采样取平均（Bootstrap）
- 报告与人类标注的 Pearson/Spearman 相关性

**界面**：
```
G-Eval Criteria
├─ 事实准确性    4.2/5  [理由: 模型正确引用了2023年数据...]  CI: [4.0, 4.4]
├─ 逻辑连贯性    3.8/5  [理由: 第二段与第三段存在跳跃...]   CI: [3.5, 4.1]
└─ 回答完整性    4.5/5  [理由: 覆盖了用户询问的所有子问题]    CI: [4.3, 4.7]

评估模型: gpt-4o | Temperature: 0 | 采样次数: 5 | 论文: Liu et al. EMNLP 2023
```

### 2.4 指标三：事实性原子验证（FACTScore）

**论文**：Min et al., *FActScore: Fine-grained Atomic Verification of Facts*, ACL 2023

**原理**：
- 将模型生成长文本分解为"原子事实"（Atomic Facts）
- 每个原子事实独立验证（用检索或知识库）
- 最终分数 = 被验证为真的原子事实数 / 总原子事实数

**适用场景**：
- 知识密集型生成（医疗、法律、金融）
- 精准识别"幻觉"发生在哪个具体事实上

**科研级要求**：
- 原子化分解的质量需要人工抽样验证
- 知识来源需要可追溯（Wikipedia / 领域知识库 / 检索结果）
- 报告 Precision / Recall / F1（事实验证本身也有误差）

**界面**：
```
FACTScore: 0.73
├─ 原子事实总数: 47
├─ 已验证为真:   34  ✅
├─ 无法验证:     9   ⚠️  [点击查看来源缺失列表]
└─ 验证为假:     4   ❌  [点击查看具体错误]

错误示例:
❌ "2023年GDP增长5.2%" → 实际为5.0% [来源: 国家统计局]
❌ "该药物由A公司研发" → 实际为B公司 [来源: FDA数据库]

论文: Min et al. ACL 2023
```

### 2.5 指标四：自一致性幻觉检测（SelfCheckGPT）

**论文**：Manakul et al., *SelfCheckGPT: Zero-Resource Black-Box Hallucination Detection for Generative Large Language Models*, ACL 2023

**原理**：
- 不依赖外部知识库（零资源）
- 对同一个问题采样多个回答，检查事实一致性
- 如果模型对同一个事实给出矛盾的答案 → 高幻觉概率

**适用场景**：
- 黑盒模型（无法访问 logits）
- 快速幻觉筛查，无需构建知识库

**科研级要求**：
- 采样次数（通常 5-10 次）
- 一致性度量方法（BERTScore / N-gram / 问答一致性）
- 阈值校准（FPR vs FNR 权衡）

**界面**：
```
SelfCheckGPT 幻觉检测
├─ 采样次数: 5
├─ 一致性阈值: 0.7
├─ 检测到的幻觉片段:
│   ├─ "该公司成立于2010年" → 5次回答中出现 2010/2008/2010/2010/2012 → ⚠️ 不一致
│   └─ "CEO是张三" → 5次回答一致 → ✅
└─ 整体幻觉概率: 0.15

论文: Manakul et al. ACL 2023
```

### 2.6 指标五：成对偏好评估（MT-Bench / Arena 方法）

**论文**：Zheng et al., *Judging LLM-as-a-Judge with MT-Bench and Chatbot Arena*, NeurIPS 2023

**原理**：
- 不是打分，而是让 Judge LLM 做 A vs B 的成对比较
- 消除位置偏见：交换 A/B 顺序测两次
- 用 Bradley-Terry 模型从成对结果推断全局排名

**这是整个评估体系的"元指标"**：
- 两个 Prompt 版本孰优孰劣，不依赖绝对分数
- 支持多个版本的循环赛排名

**科研级要求**：
- 位置偏见校正（Swap Test）
- Judge 模型自身的一致性（同一对测多次）
- 与人类判断的一致性（Kappa 系数）

**界面**：
```
成对比较结果

        v1.2    v1.3    v1.4    v1.5    ELO
v1.2     —      =       <<      <<<    1520
v1.3     =      —       <       <<     1580
v1.4     >>     >       —       =      1680  ★ Baseline
v1.5     >>>    >>      =       —      1682

<< 表示 95% 置信水平下显著优于
< 表示 90% 置信水平下显著优于
= 表示无显著差异

位置偏见校正: 已应用 (Swap Test)
Judge 一致性: Kappa = 0.82 (与人类标注高度一致)
论文: Zheng et al. NeurIPS 2023
```

### 2.7 指标六：端到端 Agent 评估（AgentEval / ToolEval）

**论文**：
- Zhu et al., *AgentBench: Evaluating LLMs as Agents*, ICML 2023
- Qin et al., *ToolLLM: Facilitating Large Language Models to Master 16000+ Real-world APIs*, NeurIPS 2023

**核心思想**：
- 不是评估单步输出，而是评估**任务是否完成**
- 轨迹评估（Trajectory Evaluation）：Agent 走的每一步是否正确、是否最优
- 工具调用链的可解释性

**指标**：
1. **任务成功率**（Success Rate）：在 N 次尝试中成功完成的比例
2. **步数效率**（Steps to Success）：达成目标所需的平均步数 vs 最优步数
3. **工具调用准确率**：选择了正确的工具 + 参数填充正确
4. **错误恢复率**：调用失败后能否自我修正（retry / 换工具 / 调整参数）
5. **轨迹偏离度**：实际轨迹 vs 最优轨迹的编辑距离

**界面**：
```
Agent 评估: task-uuid
├─ 任务成功率: 78% (39/50)  CI: [65%, 88%]
├─ 平均步数: 4.2  vs 最优: 3.0  效率: 71%
├─ 工具调用准确率: 86%  (参数正确: 92%, 工具选择正确: 94%)
├─ 错误恢复率: 67% (6/9 次失败后成功恢复)
└─ 轨迹偏离度: 0.34 (Levenshtein 标准化距离)

失败案例分析:
├─ 12% 失败: 选择了错误工具 (应该查DB却调用了API)
├─ 6% 失败: 参数错误 (日期格式不匹配)
└─ 4% 失败: 陷入循环 (重复调用同一工具)

论文: Zhu et al. ICML 2023; Qin et al. NeurIPS 2023
```

## 三、评估方法论：统计显著性是一切结论的前提

### 3.1 Bootstrap 置信区间

**原理**：
- 从 N 个 cases 中有放回地重采样 1000 次
- 每次计算指标，得到指标的分布
- 取 2.5% 和 97.5% 分位数作为 95% 置信区间

**为什么必须？**
- 假设跑了 20 个 cases，其中 18 个通过。通过率 90%。
- 但如果再跑 20 个，可能只有 15 个通过。是真变坏了，还是抽样误差？
- Bootstrap CI 告诉你："真实通过率有 95% 概率落在 [72%, 98%] 之间"

**界面呈现**：
所有指标都必须带 CI。没有 CI 的数字不允许单独出现。

### 3.2 成对 t 检验 / Wilcoxon 符号秩检验

**原理**：
- 比较两个 Prompt 版本时，不是比较平均分，而是比较**每个 case 上的差异**
- 检验："版本 A 的 case-level 得分显著高于版本 B"

**为什么必须？**
- 版本 A 平均分 82，版本 B 平均分 80。差异显著吗？
- 可能 A 在某 5 个 cases 上碾压 B，但在其他 15 个 cases 上持平——这种结构性差异比平均分更有意义
- Case-level 检验能发现"A 在复杂 case 上显著更好"这类模式

### 3.3 效应量（Cohen's d）

**原理**：
- p-value 告诉你"差异是否显著"，但不告诉你"差异有多大"
- Cohen's d = (Mean_A - Mean_B) / Pooled_SD
- d=0.2 小效应，d=0.5 中等，d=0.8 大效应

**界面**：
```
Prompt v1.5 vs v1.4 (Baseline)
├─ 平均分差异: +0.03 (v1.5: 0.82, v1.4: 0.79)
├─ 统计检验: 成对 t 检验, p = 0.003  ✅ 显著
├─ 效应量: Cohen's d = 0.68  📊 中等偏大
└─ 结论: v1.5 在统计意义上显著优于 Baseline，效应量为中等偏大
```

## 四、评估者校准：Judge LLM 不是神

### 4.1 Judge 模型的一致性

**问题**：用 GPT-4 做 judge，但它对同一对答案两次判断可能不一致。

**要求**：
- 每个成对比较至少重复 3 次
- 报告 Judge 自身的一致性（Self-Agreement Rate）
- 如果一致性 < 80%，该 Judge 不可信，需要换模型或调 prompt

### 4.2 Judge 模型的偏见

**已知偏见**（Zheng et al., NeurIPS 2023）：
1. **位置偏见**：放在前面的答案更容易被选中
2. **长度偏见**：更长的答案更容易得高分
3. **自我提升偏见**：Judge 偏爱自己家族的模型（GPT-4 偏爱 GPT 系列）

**校正方法**：
- Swap Test：同一对答案交换顺序再判一次，结果不一致则丢弃
- Length Normalization：对回答长度做标准化后再比较
- 多 Judge 集成：用 3 个不同家族的 Judge（GPT-4 + Claude + 自研）投票

### 4.3 与人类标注的对齐

**黄金标准**：
- 随机抽取 100 个 case，人工标注
- 计算 Judge LLM vs 人类的 Cohen's Kappa
- Kappa > 0.6 表示可接受，> 0.8 表示优秀

**界面**：
```
Judge 校准状态
├─ Judge 模型: gpt-4o
├─ 自一致性: 87% (3次重复中结果一致的比例)
├─ 位置偏见: 已校正 (Swap Test 通过率 94%)
├─ 长度偏见: 已校正 (长度标准化系数: 0.12)
├─ 人类对齐: Kappa = 0.74  ✅ 良好
└─ 可信度: 高  ✅
```

## 五、界面架构：统计报告，不是仪表盘

### 5.1 核心原则

- **没有绝对分数**：所有数字必须带置信区间或成对比较上下文
- **没有单独的可视化**：图表必须附带统计结论（"差异显著"或"不显著"）
- **可审计**：每个结论都可以下钻到原始数据（哪些 cases 支持这个结论）

## 六、实施路径

### Phase 1：统计基础设施（2-3 周）
- 后端：Bootstrap 置信区间计算
- 后端：成对 t 检验 / Wilcoxon 检验
- 后端：Cohen's d 效应量
- 前端：所有指标显示 CI

### Phase 2：成对比较 + ELO（2-3 周）
- 后端：成对比较 API（Judge LLM A vs B）
- 后端：Swap Test 位置偏见校正
- 后端：Bradley-Terry ELO 计算
- 前端：版本对比矩阵 + ELO 排名

### Phase 3：引用级指标（4-6 周）
- BERTScore 集成（指定模型、层、rescaling）
- G-Eval 集成（CoT + Temperature 0 + Bootstrap）
- FACTScore 集成（原子化 + 知识库验证）
- SelfCheckGPT 集成（多次采样 + 一致性检测）

### Phase 4：Judge 校准（2-3 周）
- Judge 自一致性报告
- 与人类标注的 Kappa 计算
- 多 Judge 集成投票

### Phase 5：Agent 评估（3-4 周）
- 端到端任务成功率
- 轨迹偏离度计算
- 工具调用链分析

---

## 七、一句话总结

> **科研级评估不是"多几个指标"，而是"每个结论都有统计支撑"。**
>
> 我们的工具应该产出的是**可以放进论文里的结果**——带置信区间、带 p-value、带效应量、带方法论引用。
