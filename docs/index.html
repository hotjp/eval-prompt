eval-prompt

团队级 Prompt 资产的本地私有化管理中枢。以纯 Go 二进制单文件形式分发，通过浏览器访问 Web UI，Agent 通过 CLI/MCP 协议消费。

核心特性

一站式解决 Prompt 的编写、版本控制、效果验证与团队共享

Git 版本控制

所有 Prompt 资产纳入 Git 管理，支持版本历史、Diff 对比、团队协作提交。提交信息格式化，便于追溯。

Eval 效果验证

支持 Deterministic Checker 和 LLM Rubric Grader 两种评测方式，量化 Prompt 质量，守护发布标准。

MCP 协议支持

标准 MCP 协议适配，Agent 可通过 CLI 或 MCP 接口查询、获取、触发评测 Prompt 资产。

跨模型适配

Claude ↔ GPT ↔ 本地模型自动适配，自动调整格式、few-shot 数量、温度参数，保持效果一致性。

Agent 自主优化

自动分析失败用例、生成优化建议、执行效果验证，形成人机协同的持续改进闭环。

安全沙箱

文件系统隔离、命令白名单、执行超时限制，防止路径遍历和恶意指令注入。

CLI 命令行

简洁的命令行接口，Agent 和人类均可高效操作

资产管理

$ ep asset list $ ep asset show common/code-review $ ep asset create --id common/sql-inject --file prompt.md $ ep asset edit common/sql-inject --stdin

版本与 Eval

$ ep snapshot list common/code-review $ ep snapshot diff common/code-review v1.2 v1.3 $ ep eval run common/code-review --json $ ep eval compare common/code-review v1.2 v1.3

Agent 自主优化

$ ep optimize common/code-review --strategy failure_driven $ ep eval diagnose run-123 --format json $ ep adapt common/code-review --from claude --to gpt-4o

Agent 消费

$ ep trigger match "检查Go代码SQL注入" --top 1 $ ep trigger match "代码评审" --biz-line common --json

技术栈

严选工业级组件，确保生产环境稳定可靠

connect-go + net/http

gRPC + HTTP 双模协议，标准库 HTTP，无额外框架依赖

ent + SQLite

类型安全、自动迁移、代码生成，纯 Go 实现无 CGO

go-git/v6

纯 Go Git 实现，嵌入二进制，支持 init/add/commit/diff/log

React + Ant Design + ECharts

Web UI：Monaco Editor 编辑、雷达图趋势图、数据可视化

log/slog + opentelemetry

结构化 JSON 日志，OTLP 链路追踪与指标，开箱即用
