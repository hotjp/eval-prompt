适用场景

无论你是哪种工作流，eval-prompt 都能帮你更好地管理 AI 资产

Prompt 库

集中管理团队所有 Prompt，Git 版本追溯，质量评测保障，标签分类检索。

文件 Registry

统一管理 AI 工作流中的上下文资产：示例、约束、工具描述、参考文档。

Agent Skills 库

让 Agent 有技能目录可查可用，语义匹配触发，MCP 协议原生支持。

知识库

带版本控制的知识沉淀，支持 Markdown 文件，可追溯可对比可回滚。

核心能力

围绕 AI 资产的版本化管理、质量验证与团队协作

Git 版本控制

所有 Prompt 资产纳入 Git 管理，支持版本历史、Diff 对比、团队协作提交。提交信息格式化，便于追溯。

Eval 效果验证

支持 Deterministic Checker 和 LLM Rubric Grader 两种评测方式，量化 Prompt 质量，守护发布标准。

MCP 协议支持

标准 MCP 协议适配，Agent 可通过 CLI 或 MCP 接口查询、获取、触发评测 Prompt 资产。

Agent 参考文档

跨模型适配

Claude ↔ GPT ↔ 本地模型自动适配，自动调整格式、few-shot 数量、温度参数，保持效果一致性。

Agent 自主优化

自动分析失败用例、生成优化建议、执行效果验证，形成人机协同的持续改进闭环。

安全沙箱

文件系统隔离、命令白名单、执行超时限制，防止路径遍历和恶意指令注入。

资产管理

$ ep asset list
查看所有 Prompt 资产
$ ep asset show common/code-review
查看指定资产的详细信息
$ ep asset create --id common/sql-review --file prompt.md
新建 Prompt 资产并提交 Git
$ ep asset edit common/sql-review --stdin
通过管道编辑资产内容

版本与 Eval

$ ep snapshot list common/code-review
查看资产的版本历史
$ ep snapshot diff common/code-review v1.2 v1.3
对比两个版本的差异
$ ep eval run common/code-review --json
运行评测，输出结构化结果
$ ep snapshot checkout common/code-review v1.2
切换到指定版本

Label 晋升

$ ep label list common/code-review
查看资产的所有标签
$ ep label set common/code-review prod v1.3
将 v1.3 标记为生产版本
$ ep eval compare common/code-review v1.2 v1.3
对比两个版本的评测得分
$ ep label unset common/code-review prod
移除生产标签

Agent 自主优化

$ ep optimize common/code-review --strategy failure_driven
分析失败用例，生成并验证优化
$ ep eval diagnose run-123 --format json
归因失败原因，输出优化建议
$ ep adapt common/code-review --from claude --to gpt-4o
跨模型适配 Prompt

Agent 消费

$ ep trigger match "SQL注入检测" --top 1
语义匹配最相关的 Prompt
$ ep trigger match "代码评审" --biz-line common --json
按业务线筛选，输出 JSON

同步与工具

$ ep sync reconcile
对账 Git 仓库与数据库索引
$ ep sync export --format json
导出全部资产为 JSON
$ ep version
查看版本信息

性能卓越

Go 语言实现，无 JVM、无 Node.js、无 Python 运行时

指标 eval-prompt (Go) Python/Node.js 方案
冷启动 <10ms 200-500ms
内存占用 ~16MB 80-200MB
CLI 命令 <10ms 50-200ms
二进制大小 ~28MB 依赖复杂

实测：600 个 Prompt 文件，CLI 命令执行时间 <10ms，内存占用 ~16MB

技术栈

严选工业级组件，确保生产环境稳定可靠

connect-go + net/http

gRPC + HTTP 双模协议，标准库 HTTP，无额外框架依赖

ent + SQLite

类型安全、自动迁移、代码生成，纯 Go 实现无 CGO

go-git/v6

纯 Go Git 实现，嵌入二进制，支持 init/add/commit/diff/log

React + Ant Design + ECharts

Web UI：Monaco Editor 编辑、雷达图趋势图、数据可视化

log/slog + opentelemetry

结构化 JSON 日志，OTLP 链路追踪与指标，开箱即用
